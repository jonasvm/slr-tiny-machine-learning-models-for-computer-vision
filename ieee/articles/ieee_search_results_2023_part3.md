# Suggested Rule for IEEE Screening

For this IEEE Xplore search, the following practical rule can be used.

## Keep

- Low-power computer vision
- Embedded vision systems
- Edge vision systems
- On-device vision
- Real-time vision with lightweight models
- Vision acceleration using hardware-aware methods
- CNN / ViT acceleration for constrained platforms
- Object detection, image classification, image segmentation, or object tracking with explicit efficiency constraints

## Discard

- Generic real-time computer vision without embedded, edge, or low-power constraints
- Broad sensing or 3D acquisition surveys without TinyML / Edge AI focus
- 6G/networking-heavy papers where vision is not the main constrained AI problem
- Robotics or transportation applications that use vision but do not emphasize efficient/on-device deployment
- Point-cloud-only papers without clear embedded/edge vision relevance
